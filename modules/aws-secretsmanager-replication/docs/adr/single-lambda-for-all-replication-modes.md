# Single Lambda for all replication modes

The module deploys a single Lambda function that handles EventBridge-triggered replication (Secrets Manager API calls delivered via CloudTrail), manual single-secret invocation (`secret_id`), and full-account sync (`initial_run`/`enable_full_sync`) — distinguished by event shape rather than separate functions. This reduces IAM duplication, simplifies packaging, and provides a single operational surface. The alternative (separate `automatic` and `manual` Lambdas, as in versions ≤ 1.x) was rejected because both modes share nearly all replication logic; splitting them meant maintaining two copies of the same code and two IAM policy sets.

This is a breaking change: the per-mode outputs (`lambda_automatic_replication_arn`, `lambda_manual_replication_arn`, and their `_role_arn` variants) and the `manual_replication_enabled` variable were removed in favour of the single `lambda_replication_arn` / `lambda_replication_role_arn` outputs. It is recorded here because reversing it would again change the module's output surface and IAM shape, disrupting consumers.
