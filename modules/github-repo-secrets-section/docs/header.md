# **GitHub Repository Secrets Terraform Module**

## Overview

This module manages GitHub repository secrets for **Actions**, **Codespaces**, and **Dependabot** using a single strongly-typed `config` object. It is compatible with GitHub Automated Provisioning Systems such as `ghaps` because all desired state is supplied through one top-level `config` input.

**Important**:  
The values in the `actions`, `codespaces`, and `dependabot` maps (the per-secret ciphertext strings) **must already be encrypted** using **libsodium** with the target repository's matching secret type public key.  
**Terraform does not perform any encryption**. It forwards these pre-encrypted ciphertext values to the GitHub API. Actions and Dependabot secrets are sent through the provider's `value_encrypted` argument with the matching public-key ID; Codespaces secrets continue to use the provider's non-deprecated `encrypted_value` argument.

This approach is the most secure for automated pipelines (Prefapp IDP, GitHub Actions, etc.).

## Key Features

- **Single complex object**: All secrets are defined in one `config` variable.
- **Pre-encrypted values**: Map values for each secret must be provided as libsodium-encrypted ciphertext strings.
- **Three secret types**: Actions, Codespaces, and Dependabot supported in the same module.
- **Deterministic secret updates**: When optional `*_sha256` maps are provided, the module uses `terraform_data` + `replace_triggered_by` to detect actual plaintext changes. The non-deterministic ciphertext diff is suppressed via `ignore_changes`.
- **Repository validation**: Validates `config.repository` format; callers must ensure secret names and values are set and non-empty.
- **JSON-native**: Works seamlessly with `terraform.tfvars.json` generated by external programs.

## GitHub Automated Provisioning Systems Compatibility

Expected `terraform.tfvars.json` shape:

```json
{
  "config": {
    "repository": "owner/repo",
    "actions": {
      "MY_API_KEY": "<libsodium-encrypted-actions-value>"
    },
    "actions_sha256": {
      "MY_API_KEY": "<64-char-lowercase-hex-sha256>"
    },
    "codespaces": {},
    "dependabot": {}
  }
}
```

The module maps to one Firestartr custom resource, one GitHub repository secret section, and one Terraform state. Secret map keys are GitHub secret names. Secret map values are encrypted ciphertext strings; do not pass plaintext secrets.

When `*_sha256` maps are provided alongside the secret groups (e.g. `actions_sha256` with the same keys as `actions`), the module uses `terraform_data` resources whose `input` is the SHA-256 digest. If the plaintext value changes, the digest changes, the `terraform_data` input changes, and `replace_triggered_by` triggers a replacement of the secret resource. The `ignore_changes` lifecycle rule continues to suppress spurious diffs from the non-deterministic libsodium ciphertext.

When `*_sha256` maps are absent, `terraform_data` resources have a stable `null` input, so no replacement is ever triggered. The secret lifecycle remains unchanged from the pre-fix behavior. To enable automatic updates, provide the SHA-256 digests.

## Basic Usage

### Using `terraform.tfvars.json` (recommended)

```hcl
module "repo_secrets" {
  source = "git::https://github.com/prefapp/tfm.git//modules/github-repo-secrets-section"

  config = var.config   # Terraform automatically loads terraform.tfvars.json
}
```

### Inline example

Manages GitHub secrets for **one repository only**.

```hcl
module "repo_secrets" {
  source = "git::https://github.com/prefapp/tfm.git//modules/github-repo-secrets-section"

  config = {
    repository = "prefapp/tfm"

    actions = {
      "MY_API_KEY" = "r+RFBGIn8U7z2Opm5RN7PXKdgFzefXiV91IpG3O2DrClZl9dkTJBfhRZbi2uV2nu..."
    }
    codespaces = {}
    dependabot = {}
  }
}
```
