# Plan: installed_managed_files accumulator

**Module:** `modules/github-files-set`
**Spec:** spec.md

## Changes

### `variables.tf`

Add a new top-level variable alongside `config`:

```hcl
variable "installed_managed_files" {
  description = "Accumulated list of user-managed file addresses (\"file/branch\") that have been provisioned at least once. Passed in by gh_provisioner from the entity's output secrets on every reconciliation. Defaults to [] on first apply."
  type        = list(string)
  default     = []
}
```

No validation block is added — the format of individual addresses is an operational concern of `gh_provisioner`.

### `outputs.tf`

Replace the existing `user_managed_files` output with `installed_managed_files` using `setsubtract` + `tolist`:

```hcl
output "installed_managed_files" {
  description = "Accumulated list of user-managed file addresses that have been provisioned at least once. Grows when new userManaged files are applied; shrinks when a file transitions to userManaged = false."
  value = tolist(setsubtract(
    toset(concat(
      var.installed_managed_files,
      [for f in var.config.files : "${f.file}/${f.branch}" if f.userManaged]
    )),
    toset([for f in var.config.files : "${f.file}/${f.branch}" if !f.userManaged])
  ))
}
```

**Why `setsubtract`?** It handles deduplication and subtraction in one expression and is stable across Terraform versions >= 0.14. `tolist` converts the result back to `list(string)` for a stable output type.

**Rename rationale:** `user_managed_files` described only the *current* per-apply view. `installed_managed_files` conveys the persistent, historical nature of the list. The rename is a breaking change to the output key; callers (all inside `gh_provisioner`) must be updated in the same release.
